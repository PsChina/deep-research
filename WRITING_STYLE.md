# WRITING STYLE (v5.0)

> 硬约束以 [`SKILL.md`](SKILL.md) 为准。

报告 ≠ 信息罗列,是**让读者做更好决定的工具**。3 判: 删掉这段结论会变吗? 不会→删。能告诉读者下一步做什么吗? 不能→改可操作。有证据 + cite 吗? 没有→补。**读起来像资深顾问写的分析,还是像 research assistant 拼凑的 findings dump?** 后者 → 重写。

风格参考: Bridgewater Daily Observations(据公开资料,分级锐利 + 可操作) / Economist(段落论证 + 节奏) / OpenAI DR(数据密度 + 信心标注) / McKinsey(so-what 测试)。

## 一、段落 Toulmin 6 要素 (deep 必满足 ≥4)

> Toulmin, *The Uses of Argument* (CUP 1958), Ch.III pp.87-134

| # | 要素 | 表现 | 必/可 |
|---|---|---|---|
| 1 | **Claim** | "X 是 Y" | 必 |
| 2 | **Data** | inline cite + 具体数字 | 必 |
| 3 | **Warrant** | "因为...所以" / thus | 必 |
| 4 | **Backing** | 类比 / 历史 / 原理 | 可 |
| 5 | **Qualifier** | "在 X 条件下" / "短期内" | **deep 必** |
| 6 | **Rebuttal** | "例外: 若 Y 则结论不成立" | 全文 ≥3 (deep) |

### 对照

❌ v2.2(缺 Qualifier+Rebuttal,"过度绝对"或"含糊克重"):
> 恒生科技 PE 22.08 倍(历史 24.10% 分位),处于估值洼地[7]。

✅ v2.3(6 要素):
> 恒生科技 PE 22.08 倍(历史 24.10% 分位),处于估值洼地[7]。[Backing: 历次启动前 PE <30% 分位 6 月内修复 ~70%, 2017/2020/2024 三轮回测[15]] **[Qualifier: 在海外流动性不进一步恶化前提下]**,目标 6500-7400 区间 30-40% 修复空间。**[Rebuttal: 若美联储 6 月反向加息或 IPO 加速,估值底可能延迟,2022-Q4 即先例]**

→ 同长度,后者"敢下判断 + 承认例外"的层次感。

## 一b. 段落 4 项 baseline (fast/standard 简化版)

每段满足 ≥3: ① 主张句 ② inline cite ③ 服务 sub_Q ④ so-what。

## 二、锐利度梯度

| 位置 | 锐利度 | hedge 配额 |
|---|---|---|
| **TL;DR** | **极锐利** | **0** |
| 每节主张句 | 较锐利 | ≤1 |
| 段落细节 | 中等 | 用 "在 X 条件下" 代替 "可能" |
| 风险章节 | 较锐利 | 概率或频率 |
| **信息缺口** | **明确克制** | "未找到 X → 影响 Y 判断" |

❌ "恒科估值低,可能反弹,但有不确定性,请谨慎决策。"

✅ "TL;DR: 恒科 PE 22 倍处于洼地, **但 7 项启动信号 2✅/3◯/2⚠️ — 是震荡修复,不是全面启动**[13][7]。建议利用 4800-5100 区间定投,锚定 4 触发器升级。信息缺口: 阿里 Q2 业绩共识未深挖, **没有这个数据,中期判断置信度只到 MEDIUM**。"

## 三、推理强度标注

核心 claim(TL;DR + 章节小结)必标 2 维:

| reasoning_type | reasoning_strength |
|---|---|
| `[direct_evidence]`(默认可省) | `[必然]` |
| `[推断]` | `[高度可能]` >70% |
| `[类比 YYYY]` | `[合理推测]` 40-70% |
| `[N 家共识]` | `[推测]` <40% |
| `[假设性]` | |

例: 美联储 6 月降息预期降温[3] **[direct_evidence]** → 短期美元利率维持高位 **[推断+高度可能]** → 港股科技估值修复弹性受限 **[类比 2022-Q4+高度可能]**

> 概率分级语言可见 Ray Dalio *Principles* "likelihood/probability" + LinkedIn 公开文章 "If X, then likely Y because Z"(Bridgewater 内部 conviction tag 具体格式未公开)。

## 四、章节结构

1. **Action Title** (动词/数字/方向/具体名词 ≥2)
2. **开场白** ("本节回答 X")
3. **主体** (Toulmin 6 要素)
4. **小结** ("→ 对你意味着什么:" 1-3 句)

## 五、Action Section (末尾必含)

每条"下一步" 3 要素: **触发条件** (可观察信号) + **响应动作** + **timeline**。

❌ "投资者可根据市场变化调整策略。"

✅ "未来 60 天 4 信号: ① 美联储 6 月议息(6/12)→ 降 25bp 升级配置;② 阿里 Q2 业绩(8/15)→ 云 YoY >18% 加仓 / <12% 减仓;③ 南向月度回升 800 亿+ 持续 2 月 → 启动 4/7;④ 港股 IPO 单月募资 <500 亿 → 流动性瓶颈缓解。"

## 六、视觉锚密度

每 800-1200 字 ≥1 视觉锚(决策矩阵 / 时间线 / 检查表 / 数据矩阵 / 优先级矩阵 / 流程图 / 置信度分布表)。

## 七、段落节奏

短(1-3 句)与长(4-8 句)交替。**禁止 >10 句段** → 拆 2 段或 1 段 + 表。短段是路标,长段是地形。

## 八、TL;DR 专项

3-5 bullet,每条: ① 置信度 ✅HIGH/◯MEDIUM/⚠️LOW ② 核心数字/方向/状态(判断,非描述) ③ inline cite [N] ④ (可选)推理强度。

末尾 1 句 **"以下任一发生 → 重新评估"** + 2-3 触发器。

❌ "✅ A 股 -0.05% / ✅ 恒科 -1.70% / ✅ 美联储 6 月预期反复"(罗列无判断)

✅ "✅[HIGH] A 股结构性分化,主线没撤但估值临界(电子 PE 98% 分位)[5] — **追主线赔率正在下降**"

## 九、信息缺口"克制感"

❌ "由于数据有限,本报告可能存在一些不确定性。"

✅ "本报告 3 项关键数据缺口:① 阿里 Q2 业绩共识 — 未深挖,中期判断置信度只到 MEDIUM。② 美联储 6/12 议息实时定价 — 我抓的是 5 月初快照,无法替你决定 6/12 怎么做。③ 中美关税清单最新 — 未单独搜,我承认这个盲点。"

诚实标注"我不知道什么 + 我的不知道意味着什么" = 报告可信度核心。

## 十、合规

- **投资**: "不构成投资建议" / 区分事实 vs 判断 / 给观察点不给买卖 / CFA 5 要素(decide+investment)
- **医疗**: "不构成医疗建议" / 同行评审来源
- **法律**: "不构成法律建议" / 司法管辖区
- **技术选型**: 时效标注 / 官方 vs 社区区分

## 十一、tier 启用

- **fast**: §1b + §二
- **standard**: + §四 + §五 + §十二(顾问级叙事, v3.1)
- **deep / museum**: **全部强制** — §一 Toulmin 6 + §二-§十 + §十二-§十六(顾问级)。违反 → `writing_style_check: fail`

## 十二、Central Thesis — 一句话核心判断 (v3.1, 全 tier)

> 报告不是 findings 的集合，是一个核心判断的展开论证。

每份报告必须在 TL;DR 之后、正文之前，写一行 **Central Thesis**：

```
> **Central Thesis**: <一句话核心判断 — 包含 因果关系 + 方向 + 边界条件>

❌ "本报告分析了 monorepo 和 multi-repo 的优劣"
✅ "对 2 人创业公司，monorepo 是正确方向，但前提是代码共享率 >30% 且产品方向已稳定 — 否则单体应用更务实"
```

**检验**: 删掉 Central Thesis 后，报告的每一节是否仍然有意义？如果某节和 Thesis 无关 → 该节是拼凑，删或重写。

## 十三、顾问级叙事结构 (v3.1, standard+)

> 报告 = 论点驱动的分析，不是按 sub_Q 排序的 findings dump。

### 三幕结构

```
幕一: REFRAME — "你问的是 X，但你真正需要回答的问题是 Y"
  → 重新定义问题边界，展示你对这个领域的判断力
  → 不是复述用户的问题，是升级它

幕二: ANALYSIS — "证据说什么，以及它实际意味着什么"
  → 不是按 sub_Q 罗列 findings
  → 而是按论证逻辑链组织：前提 → 证据 → 推断 → 结论
  → 反方观点自然出现在论证流中，不是单独一节

幕三: DECISION FRAMEWORK — "你应该怎么做，在什么条件下改变主意"
  → 不是给一个答案，是给一个决策系统
  → 含触发器、时间线、责任人（即使责任人是"你"）
```

### 反"拼凑"规则

| ❌ 拼凑感 | ✅ 顾问感 |
|---|---|
| "根据 Q1 的研究…根据 Q2 的研究…" | 不出现 sub_Q 编号。按论证逻辑链组织 |
| 每节独立，互不引用 | 每节至少引用 1 处前文的发现或结论 |
| "如上所述" / "前文提到" | 重述关键点（1 句），不让读者回翻 |
| 矛盾放在单独"争议"章节 | 矛盾出现在论证流中，当场标注并处理 |
| DA 发现全堆在"局限与反方观点" | DA severe 发现直接融入正文（降级/修正/反驳），moderate 才放末尾 |
| 信息缺口堆在末尾一节 | 关键缺口在相关段落内当场标注（"这个判断的前提是 X，但我们没有 X 的数据"） |
| TL;DR 是正文的摘要 | TL;DR 是独立可读的决策备忘录（不依赖正文也能用） |

### 交叉引用规则

- 每节至少引用 1 处其他节的发现（"如 §3 所述，X 的采用率…"）
- 报告末尾的"下一步"必须回引正文中的具体发现（"基于 §4 的风险登记表，优先监控信号①…"）
- 禁止孤岛节 — 任何节如果和其他节无关联，要么合并要么删除

## 十四、顾问语气校准 (v3.1)

> 语气在"学术论文"和"推特帖子"之间。权威但不傲慢，直接但不武断。

### 语气对照

| ❌ 学术/被动 | ❌ 过于随意 | ✅ 顾问语气 |
|---|---|---|
| "研究表明 X 与 Y 呈正相关" | "X 和 Y 基本上是同一个东西" | "X 驱动 Y — 5 个独立来源一致确认这个方向[1][3][7]" |
| "可以认为 monorepo 可能较为合适" | "monorepo 就对了不用想了" | "我们推荐 monorepo，但在 §5 列了 3 个触发回退的条件" |
| "根据现有文献综合分析" | "我搜了一圈发现" | "基于 20 个来源的交叉验证，核心结论如下：" |
| "存在一定的不确定性" | "说实话我也不确定" | "这个判断的置信度是 MEDIUM — 要提高需要 X 数据（见 §8 信息缺口）" |

### 顾问专属技法

1. **先给答案再解释** — 每节第一句就是结论，不铺垫
2. **用"我们"不用"本报告"** — "我们推荐"比"本报告建议"更直接
3. **数字给锚点** — "X 占 34%"后面跟"对比行业平均 22%[5]"（数字孤立的=没有信息量）
4. **不确定时说"要知道什么才能确定"** — 不是"我们不确定"，是"要确定需要 X"
5. **不要 process，要 insight** — 不写"我们搜索了 5 个数据库"，写"5 个数据库一致显示 X"

## 十五、Opening Hook — 开篇 3 句话定调 (v3.1, standard+)

> 读者在 TL;DR 后、正文前，用 3 句话判断要不要继续读。

```
模板:
1. 为什么现在要关心这个问题?(urgency — 趋势/事件/数据点)
2. 为什么大多数人的直觉是错的?(tension — 常见误区/意外发现)
3. 本报告的独特视角是什么?(positioning — 一句话区分度)

❌ "随着微服务架构的普及，monorepo vs multi-repo 成为一个常见问题。本文将从多个角度分析…"
✅ "2026 年，Turborepo 月下载量突破 200 万[2]，monorepo 已从小众选择变成 JS/TS 生态的默认项。但 2024 年 Faros 对 320 个团队的基准测试显示 monorepo PR 周期反而慢 9.5 倍[7] — 工具成熟了，实践跟上了吗？本报告的独特视角：我们不是在比较两种 repo 策略，而是在帮你算清'你的 2 人团队到底该为哪种隐形成本买单'。"
```

## 十六、顾问级 TL;DR 升级 (v3.1)

> TL;DR 不是摘要 — 是独立可读的决策备忘录。读者只读 TL;DR 就应该能做决定。

在原 TL;DR 结构上增加：

```
## TL;DR

> **Central Thesis**: <一句话>

**如果你只记 3 件事**:
1. <最反直觉的发现> — <为什么重要>
2. <最大风险> — <缓解成本>
3. <第一个行动> — <今天下班前能做>

<原 TL;DR bullets 保留>

⚠️ **何时改主意**: ① … ② … ③ …
```

### 顾问级 TL;DR 检验
- [ ] 删掉正文，TL;DR 能独立支撑一个决策吗？
- [ ] 每句话有 cite 吗？
- [ ] hedge word = 0 吗？
- [ ] "如果你只记 3 件事"有意外感吗（不是众所周知的事实）？
